# 前言

GPUI 是一个来自 Zed 编辑器的 Rust GUI 框架。它使用 GPU 加速渲染，采用混合的保留/立即模式架构，通过 Entity-View-Element 三层模型组织界面。

本书的目标读者是有 Rust 经验的开发者。我们不会教你 Rust 基础——我们假设你已经熟悉所有权、trait、泛型和异步编程。我们也不会花大量篇幅讲 "什么是 GUI" 这种通用概念。

这本书的风格是代码先行。每个章节从具体可运行的代码开始，然后解释背后的原理。你会看到大量 `gpui::*` 的示例，从最简 Hello World 到自定义 Element、异步数据加载、虚拟列表。

## 为什么写这本书

GPUI 的文档很少。它不是像 egui 或 Iced 那样面向公众的框架——它最初只是 Zed 编辑器的内部渲染引擎。但正是因为它被一个生产级编辑器打磨过，它的设计思路与大多数 GUI 框架不同：

- **GPU 优先**：所有渲染走 GPU，不做 CPU 端的像素操作
- **单一所有者**：`App` 拥有所有状态，通过 Entity 句柄访问
- **混合模式**：View 层是声明式的，Element 层是命令式的
- **Action/Keymap 系统**：内置完整的快捷键与命令体系

这些设计让 GPUI 适合做高性能编辑器，也适合任何需要精细控制渲染的 GUI 应用。

## 本书结构

**第一部分（入门）**：回答 "GPUI 是什么"，帮你把环境搭好，写出第一个能跑的窗口。

**第二至四部分（基础、Views、Elements）**：Entity 系统、Context 类型、数据流、Render trait、div 容器、文本和图片。这是日常开发的核心。

**第五至七部分（布局、样式、事件）**：Flexbox、Grid、Styled trait、鼠标和键盘事件、Action/Keymap 系统。

**第八至十部分（焦点、异步、状态）**：FocusHandle、Task 系统、Observe/Subscribe/Global 三种状态模式。

**第十一至十三部分（高级元素、文本深入、渲染管线）**：虚拟列表、弹出层、动画、字体管理、Scene 和 GPU 渲染。

**第十四至十六部分（平台、测试、调试）**：macOS/Linux 后端、`#[gpui::test]`、Inspector、性能分析。

**第十七部分（Cookbook）**：计数器、Todo、文本编辑器、图片画廊等完整应用。

## 版本说明

本书基于 GPUI 最新源码编写。GPUI 的 API 仍在演进，部分细节可能与你的版本有出入。如果你发现不一致，建议直接查阅 GPUI 源码中对应模块——框架本身就是最好的文档。
